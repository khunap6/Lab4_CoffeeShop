# คำถามท้ายบท (Review Questions) LAB6
Sequelize คืออะไร และมีบทบาทสำคัญอย่างไรในการช่วยให้เราเขียนโค้ด JavaScript เพื่อสั่งงานฐานข้อมูลได้ง่ายขึ้น? 
-  Sequelize คือเครื่องมือประเภท ORM (Object-Relational Mapping) ซึ่งเป็น Node package ที่ทำหน้าที่จัดการฐานข้อมูลของเรา มีบทบาทสำคัญคือ
    - ลดการเขียนโค้ด SQL: ช่วยให้เราสามารถเขียนโค้ด JavaScript เพื่อสั่งงานฐานข้อมูลโดยตรง
    - การจัดการฐานข้อมูล: เป็น package สำหรับจัดการฐานข้อมูลที่ช่วยในการสร้างฐานข้อมูลและทำงาน CRUD ได้

ไฟล์ .sqlite ที่เกิดขึ้นมาทำหน้าที่อะไร?
- ทำหน้าที่เป็น ฐานข้อมูล (Database) เก็บข้อมูลทั้งหมดของเราในรูปแบบไฟล์ ทำให้เคลื่อนย้ายสะดวกและไม่ต้องรัน Database Server แยก

คำสั่ง sequelize.sync({ force: false }) มีความสำคัญอย่างไร หากเราเปลี่ยนเป็น true จะเกิดอะไรขึ้น?
- สร้างตารางจาก Model ถ้ายังไม่มี
force: false → ไม่ลบข้อมูลเดิม
force: true → ลบตารางเดิมแล้วสร้างใหม่ ทำให้ ข้อมูลทั้งหมดหาย

ในฟังก์ชัน create ตัวแปร req.body ได้ข้อมูลมาจากส่วนใดของ HTTP Request?
-req.body ได้ข้อมูลมาจาก Body ของ HTTP Request เช่น JSON ที่ส่งมาจาก Client ผ่าน POST หรือ PUT Request

อธิบายหลักการทำงานของคีย์เวิร์ด async และ await ที่ถูกนำมาใช้ในฟังก์ชันของ UserController.js ว่าเกี่ยวข้องกับการทำงานกับฐานข้อมูลอย่างไร?
-async ทำให้ฟังก์ชันรองรับการทำงานแบบไม่พร้อมกัน (asynchronous)
await ใช้รอผลลัพธ์จากคำสั่งฐานข้อมูล เช่น findAll() หรือ create()
ช่วยให้โค้ดเขียนง่ายเหมือนทำงานทีละขั้น และรอข้อมูลจากฐานข้อมูล


คำถามท้ายบท LAB7
1.เหตุใดผู้เขียนจึงระบุว่า Vue.js เป็นทางเลือกที่น่าสนใจเมื่อเปรียบเทียบกับ React หรือ Angular โดยพิจารณาจากขนาดและโครงสร้างภาษา?
-เพราะ Vue.js มีขนาดเล็ก โครงสร้างภาษาอ่านง่าย เรียนรู้เร็ว ไม่ซับซ้อน เหมาะทั้งมือใหม่และโปรเจกต์ขนาดเล็ก–กลาง เมื่อเทียบกับ Angular ที่โครงสร้างใหญ่ และ React ที่ต้องพึ่งไลบรารีเสริมหลายตัว

2.การใช้งาน Vue.js ผ่าน CDN มีข้อดีหลัก ๆ ที่สำคัญต่อการเริ่มต้นใช้งานและการโหลดเว็บไซต์อย่างไรบ้าง?
-เริ่มใช้งานได้ทันที ไม่ต้องติดตั้งเครื่องมือเพิ่มเติม
-โหลดเร็ว ลดความซับซ้อนในการตั้งค่า
-เหมาะกับเว็บขนาดเล็กหรือการทดลองเรียนรู้

3.ในโปรเจกต์ Vue.js ที่สร้างด้วย Vite ไฟล์ index.html ถูกเรียกว่า "ประตูหน้าบ้าน" (Entry Point) และมีบทบาทสำคัญในการทำงานของ Single Page Application (SPA) อย่างไร?
-index.html เป็นจุดเริ่มต้นของแอป ทำหน้าที่โหลด JavaScript หลัก และเป็นที่ที่ Vue นำคอมโพเนนต์ทั้งหมดมาแสดงในหน้าเดียวของ SPA

4.จงอธิบายขั้นตอนสำคัญ 3 ขั้นตอนที่เกิดขึ้นในไฟล์ src/main.js ซึ่งถือเป็น "จุดควบคุมหลัก" (Bootstrap File) ในการเริ่มต้นแอปพลิเคชัน Vue 3
-import Vue และ App
-สร้างแอปด้วย createApp()
-นำแอปไปแสดงบนหน้าเว็บด้วย mount()

5.เมื่อใช้ Vue Router, แท็ก <router-view /> ในไฟล์ src/App.vue ทำหน้าที่อะไร และมีความสำคัญอย่างไรต่อการจัดการหน้าเว็บ?
-เป็นตำแหน่งแสดงหน้า (Component) ตาม URL ที่เปลี่ยนไป ช่วยให้เว็บเปลี่ยนหน้าได้โดยไม่ต้องรีเฟรชหน้าใหม่ ทำให้เป็น SPA อย่างสมบูรณ์
